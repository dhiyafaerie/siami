## Project Overview

**SIAMI** (Sistem Informasi Audit Mutu Internal) is a Laravel 11 + Filament 3 admin panel for managing Internal Quality Audits (AMI) at a university. The UI and domain terminology are in Indonesian.

- **Stack:** Laravel 11, PHP 8.2+, Filament 3, filament-shield (RBAC), filament-tinyeditor (rich text), MySQL, Spatie ActivityLog, DomPDF, Maatwebsite Excel, Vue 3 (landing page)
- **Local environment:** MAMP, database `db_siami`, accessible at `https://127.0.0.1/admin`
- **Public landing page:** `https://127.0.0.1/` — Vue 3 SPA with public stats API

## Common Commands

```bash
# Run development server (with queue, logs, and Vite)
composer dev

# Run migrations
php artisan migrate

# Run a single test
php artisan test --filter TestName

# Regenerate Filament shield permissions
php artisan shield:generate --all

# Upgrade Filament assets after package updates
php artisan filament:upgrade
```

## Architecture

### Filament Admin Panel

Everything lives under the single `admin` panel (`AdminPanelProvider`). Resources, pages, and widgets are auto-discovered from their respective `app/Filament/` subdirectories. The panel uses `filament-shield` for role-based access control — permissions and policies are auto-generated per resource. Database notifications are enabled (`databaseNotifications()`).

### Domain Model & Relationships

The audit workflow flows like this:

```
Cycle (audit year/period)
  └── Standard (audit criteria, belongs to a Cycle)
        ├── Prodiattachment (evidence submitted by a Prodi user per Standard)
        ├── Auditscore (score 1–4 given by an Auditor per Standard + Prodi)
        └── Nonconformity (KTS findings per Standard + Prodi)

Faculty
  └── Prodi (study program, belongs to Faculty, linked to a User account)
        ├── Prodiattachment
        ├── Auditscore
        └── Nonconformity

User (roles: super_admin, auditor, faculty-based roles)
  └── Auditor (maps a User to an auditor role)
```

Key conventions:
- Foreign keys use the **plural table name** as the column prefix: `cycles_id`, `prodis_id`, `standards_id`, `users_id`, `auditors_id` (auditors_id references the `users` table).
- `Cycle` model has `is_locked` and `is_active` boolean casts; `booted()` hook ensures only one active cycle at a time. `CycleResource` has a `toggle_lock` action.
- `Auditscore.score` is an integer 1–4 with the mapping: 1 = Kurang, 2 = Cukup, 3 = Baik, 4 = Sangat Baik.
- All models use `$guarded = []` instead of `$fillable`.
- Prodi users are identified by `$user->prodi->isNotEmpty()` relationship, NOT by a 'prodi' role.
- Fakultas users are identified by `$user->faculty !== null`.

### Multi-Keyword Standards

Standards can have multiple sub-items (A., B., C.) with comma-separated keywords. When `count(keywords) > 1`:
- **Deskriptor** is split at A./B./C. markers via `preg_split('/\s*(?=[B-Z]\.\s)/', ...)`
- **Filament tables**: sub-items separated by `<hr>` (no rowspan in Filament)
- **Excel exports**: separate rows with merged cells (No, Prodi, Nilai, Catatan columns)
- **PDF export**: HTML `rowspan` for merged columns
- Use `Standard::htmlToPlainText()` instead of `strip_tags()` to preserve line breaks from HTML block elements

### KTS (Ketidaksesuaian) Workflow

KTS is split across three pages by role:

| Role | Page | Function |
|---|---|---|
| Super Admin | NonconformityResource (Master KTS) | CRUD kode KTS + kategori (Select: Mayor/Minor/Observasi) |
| Auditor | AuditorKts page | View assigned KTS, edit detail, verifikasi & tutup, buka kembali |
| Auditor | StandardsTable (Input KTS action) | Pick master KTS or create new, fill description + prodi + deadline |
| Prodi | ProdiKts page | View KTS, ajukan tindakan perbaikan |

KTS status workflow: `terbuka` → `dalam_perbaikan` → `ditutup`

Kategori options are defined as enum in `Nonconformity::kategoriOptions()`: Mayor, Minor, Observasi. Color badges via `kategoriColor()`.

The `Nonconformity` model has status constants (`STATUS_TERBUKA`, `STATUS_DALAM_PERBAIKAN`, `STATUS_DITUTUP`) and status check methods (`isTerbuka()`, `isDalamPerbaikan()`, `isDitutup()`). Uses `LogsActivity` trait for audit logging.

### Role-Based Behavior

The `StandardsTable` page (Tabel Kompilasi) is the most complex page — it renders differently based on the authenticated user's role:
- **Prodi users** (`$user->prodi->isNotEmpty()`): see only their own attachments, can submit/edit evidence. Program Studi field shown but disabled.
- **Auditor users** (`$user->hasRole('auditor')`): see "Nilai Audit" column and "Input Nilai"/"Input KTS" actions; can filter by Prodi.
- **Fakultas users** (`$user->faculty !== null`): see data for prodis in their faculty.
- **Super Admin**: sees everything.

### Analytics Page

`Analitik.php` is a dashboard page accessible to super_admin, auditors, and faculty users (not prodi). It displays chart widgets registered as Livewire components in `AppServiceProvider`:
- **KelengkapanProdiChart** — bar chart of evidence completion % per prodi (active cycle)
- **TotalSkorProdiChart** — bar chart of total audit scores per prodi (15-color palette)

Both widgets use the `ScopesAnalitik` trait for role-based prodi filtering (super_admin sees all, faculty sees their prodi, auditors see prodi they've audited).

### Export Capabilities per Role

| Role | Excel | PDF |
|---|---|---|
| Auditor | StandardsTableExport (their audit data) | Per prodi (ProdiAuditPdfExport) |
| Fakultas | FakultasRekapExport (their faculty, optional prodi filter) | Per prodi |
| Super Admin | FakultasRekapExport (pick fakultas, prodi, or all) | Per prodi |
| Any (via URL) | — | LaporanAmiPdfExport (full AMI report with cover, standards eval, KTS table) |

### Navigation Groups

Filament resources are grouped under `"AMI"` in the sidebar. Label/title strings are in Indonesian (e.g., "Standar", "Master KTS", "KTS Auditor", "KTS Prodi").

### Public Landing Page

A Vue 3 SPA served at `/` (`resources/views/landing.blade.php` + `resources/js/landing/app.js`). Fetches aggregated stats from `/api/public/stats` (`PublicStatsController`) which returns prodi count, faculty count, cycle count, auditor count, and per-cycle standards counts. Built with Vite.

### Custom Page: StandardsTable

`app/Filament/Pages/StandardsTable.php` implements `HasTable` directly (not a Resource). It uses `updateOrCreate` for both Prodiattachment and Auditscore submissions — re-submitting overwrites the previous entry scoped to the authenticated user.
