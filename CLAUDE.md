## Project Overview

**SIAMI** (Sistem Informasi Audit Mutu Internal) is a Laravel 11 + Filament 3 admin panel for managing Internal Quality Audits (AMI) at a university. The UI and domain terminology are in Indonesian.

- **Stack:** Laravel 11, PHP 8.2+, Filament 3, filament-shield (RBAC), filament-tinyeditor (rich text), MySQL
- **Local environment:** MAMP, database `db_siami`, accessible at `https://127.0.0.1/admin`

## Common Commands

```bash
# Run development server (with queue, logs, and Vite)
composer dev

# Run migrations
php artisan migrate

# Run a single test
php artisan test --filter TestName

# Regenerate Filament shield permissions
php artisan shield:generate --all

# Upgrade Filament assets after package updates
php artisan filament:upgrade
```

## Architecture

### Filament Admin Panel

Everything lives under the single `admin` panel (`AdminPanelProvider`). Resources, pages, and widgets are auto-discovered from their respective `app/Filament/` subdirectories. The panel uses `filament-shield` for role-based access control — permissions and policies are auto-generated per resource.

### Domain Model & Relationships

The audit workflow flows like this:

```
Cycle (audit year/period)
  └── Standard (audit criteria, belongs to a Cycle)
        ├── Prodiattachment (evidence submitted by a Prodi user per Standard)
        └── Auditscore (score 1–4 given by an Auditor per Standard + Prodi)

Faculty
  └── Prodi (study program, belongs to Faculty, linked to a User account)
        ├── Prodiattachment
        └── Auditscore

User (can have role: auditor, prodi, admin, etc.)
  └── Auditor (maps a User to an auditor role)
```

Key conventions:
- Foreign keys use the **plural table name** as the column prefix: `cycles_id`, `prodis_id`, `standards_id`, `users_id`, `auditors_id` (auditors_id references the `users` table).
- `Auditscore.score` is an integer 1–4 with the mapping: 1 = Kurang Cukup, 2 = Kurang, 3 = Cukup, 4 = Sangat Cukup.
- All models use `$guarded = []` instead of `$fillable`.

### Role-Based Behavior

The `StandardsTable` page (Tabel Kompilasi) is the most complex page — it renders differently based on the authenticated user's role:
- **Prodi users** (`$user->prodi->isNotEmpty()`): see only their own attachments, can submit evidence via "Input Dokumen" action.
- **Auditor users** (`$user->hasRole('auditor')`): see a "Nilai Audit" column and "Input Nilai" action; can filter by Prodi and submit scores.
- **Admin**: sees everything without filters.

### Navigation Groups

Filament resources are grouped under `"AMI"` in the sidebar. Label/title strings are in Indonesian (e.g., "Standar", "Ketidaksesuaian", "KTS").

### Custom Page: StandardsTable

`app/Filament/Pages/StandardsTable.php` implements `HasTable` directly (not a Resource). It uses `updateOrCreate` for both Prodiattachment and Auditscore submissions — re-submitting overwrites the previous entry scoped to the authenticated user.
